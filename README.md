# AI-Driven SOC Alert Prioritization Framework

Modern Security Operations Centers (SOCs) are overwhelmed by "alert fatigue"—a high volume of security alerts generated by various tools (SIEM, IDS, EDR) that makes it difficult for analysts to identify and respond to critical threats in time. 

This project provides an end-to-end, AI-driven framework and interactive dashboard designed to solve this problem. By leveraging network packet sniffing and machine learning (XGBoost/Ensemble models), the system analyzes, scores, and prioritizes security alerts in real time, allowing security teams to focus on the most critical threats first.

![ROC Curve - Attack Detection](assets/Screenshot_2026-05-06_144246.png)  
*ROC Curve comparing the predictive power of Logistic Regression, XGBoost, and the production Random Forest model.*

The framework evaluated Logistic Regression, XGBoost, and Random Forest against a strict 20% holdout test set (123,475 flows).

| Machine Learning Classifier | Overall Accuracy | Macro Precision | Macro Recall | False Positive Rate (FPR) |
| :--- | :--- | :--- | :--- | :--- |
| Logistic Regression (Baseline) | 98.21% | 0.88 | 0.86 | 0.17% |

The Random Forest classifier emerged as the definitive champion. By utilizing ensemble bagging (100 independent decision trees), the model effectively neutralized variance and achieved a state-of-the-art 99.58% classification accuracy, representing a 99.8% reduction in false positive generation compared to legacy systems.

## 🚀 Quick Start (TL;DR)

Get the SOC dashboard running locally in 4 commands:

```powershell
# 1. Navigate to the project directory
cd d:\soc

# 2. Install all required dependencies
pip install -r requirements.txt

# 3. Start the API backend and dashboard server
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000

# 4. Open your browser and navigate to:
 http://localhost:8000

```

## ✨ Key Features & Dashboard Walkthrough

The web-based dashboard is the control center for the SOC framework, featuring 6 core tabs:

1. **📊 Dashboard:** The high-level executive view. Displays Key Performance Indicators (KPIs), active alert summaries, and real-time threat scores.
2. **🚨 Alerts:** The prioritized alert queue. Lists anomalies scored by the ML model, sorted by risk priority to combat alert fatigue. Analysts can view details, severity, and timestamps.
3. **📈 Metrics:** Deep dive into system health. Visualizes network statistics, packet capture rates, model performance (precision/recall), and throughput.
4. **🖥️ Live Monitor:** Real-time visibility. Provides a live stream of processed packets, Scapy sniffing activity, and instantaneous anomaly detection logs.
5. **🔄 Traffic Simulation:** Testing and validation. Allows analysts to inject simulated network traffic scenarios (e.g., normal flow vs. DDoS/Brute Force attacks) to validate model detection capabilities.
6. **🧠 Analytics:** Post-incident review. Offers historical data analysis, trend visualization, and correlation exploration for proactive threat hunting.

## 🏗️ Architecture & Data Flow

The framework operates on a continuous pipeline from raw network data to actionable intelligence:

```text
       Raw Network Traffic
               │
               ▼
      [ Scapy Sniffer ] ───────── (Captures & preprocesses packets)
               │
               ▼
   [ Feature Engineering ] ────── (Extracts ML-ready features from PCAP data)
               │
               ▼

               │
               ▼
```

## 🛠️ Prerequisites

Before running the system, ensure you have the following installed:

* **Python 3.8+**: Ensure Python is added to your system PATH.
* **Npcap**: **CRITICAL for Windows users.** Required for Scapy to perform real-time packet sniffing.
* **Administrator Privileges**: Packet sniffing requires elevated permissions. Run your terminal/PowerShell as an Administrator.
* **Dataset (Optional)**: If you intend to retrain the model, you will need a dataset (e.g., CIC-IDS 2017) formatted in CSV.

* **ReDoc:** `http://localhost:8000/redoc`

**Example Endpoints:**

* `GET /api/packets` — Retrieve recently processed network packets.
* `GET /api/alerts` — Fetch the prioritized list of active alerts.
* `POST /api/simulate` — Trigger a simulated network attack.
* `POST /api/retrain` — Initiate model retraining on new data.

## 🏋️ Model Retraining Guide

To maintain accuracy against evolving threats, the machine learning model can be retrained with new incident data.

1. **Prepare Data:** Ensure your new dataset contains the required network features and is labeled correctly (e.g., `0` for benign, `1` for malicious).
2. **Execute Retraining Script:**
```powershell
python models/retrain.py --data path/to/new_dataset.csv
```




3. **Validation:** The script will automatically output new precision, recall, and F1-scores, and save the updated `.xgb` or `.pkl` model file to the `models/` directory.
4. **Restart:** Restart the Uvicorn server to load the fresh model into memory.

## 🐛 Troubleshooting

| Problem | Possible Cause | Solution |
| --- | --- | --- |
| **Scapy sniffing errors** | Missing Npcap or lack of permissions. | Install Npcap. Run PowerShell as **Administrator**. |
| **Uvicorn not recognized** | Dependencies not installed / PATH issue. | Run `pip install -r requirements.txt`. Try `python -m uvicorn`. |
| **No alerts generating** | Model needs tuning or testing on benign network. | Use the **Traffic Simulation** tab to inject test anomalies. |
| **API Connection Refused** | Server isn't running or port conflict. | Ensure `uvicorn` is running. Check if port `8000` is free. |
